Concepto de Software

El software es el conjunto de programas, aplicaciones y datos que permiten a una computadora o dispositivo realizar tareas específicas, mientras que el Hardware, es el conjunto de componentes físicos y tangibles que forman un dispositivo electrónico, como computadoras, teléfonos móviles o electrodomésticos. Incluye elementos como el procesador (CPU), la memoria RAM, el disco duro, la placa base, etc..

En esto tenemos el Software de aplicación, que son los programas diseñados para realizar tareas específicas para el usuario, como procesadores de texto, navegadores web, aplicaciones de diseño, juegos, etc. Luego tendríamos el Software de sistema que incluye los programas que permiten que el hardware funcione correctamente, como los sistemas operativos (por ejemplo, Windows, macOS, Linux), controladores de dispositivos, y utilidades del sistema, y por último el Software de desarrollo que son las herramientas que permiten a los programadores crear otros programas, como lenguajes de programación (Java, Python, C++) y entornos de desarrollo integrados (IDEs).

El hardware se clasifica en tres tipos principales. El hardware interno incluye componentes como el procesador (CPU), memoria RAM, disco duro o SSD, placa base y tarjetas de expansión, esenciales para el funcionamiento interno del sistema. El hardware externo abarca dispositivos periféricos de entrada (teclado, ratón), salida (monitor, impresora) y entrada/salida (pantallas táctiles, discos externos) que permiten la interacción con el usuario. Por último, el hardware de comunicación incluye tarjetas de red, routers, módems y switches, que facilitan la conexión con otros dispositivos y redes.

Resumen
Tanto el software como el hardware son esenciales para el funcionamiento de los dispositivos electrónicos, ya que da instrucciones precisas sobre cómo debe ejecutarse cada operación o proceso.

Conceptos de Código Fuente, código Objeto, código Ejecutable y tipos de traductores.


El código fuente es el conjunto de instrucciones escritas por el programador en un lenguaje de programación de alto nivel, como C++, Java o Python. Este código es comprensible por los seres humanos, pero no puede ser ejecutado directamente por la computadora. Para que el código fuente funcione, debe ser traducido a un formato que la máquina pueda entender.


El código objeto es el resultado de compilar parcialmente el código fuente. Es una representación en lenguaje de bajo nivel o código máquina, que aún no está completamente listo para ser ejecutado, ya que puede requerir enlaces adicionales.


El código ejecutable es el producto final de la compilación y el enlace; es un archivo binario que la computadora puede ejecutar directamente, ya que está completamente traducido a instrucciones específicas de la arquitectura de la máquina.


Para convertir el código fuente en código ejecutable, se utilizan diferentes tipos de traductores:


  • Un compilador traduce todo el código fuente a código objeto en una sola pasada, generando un archivo independiente y optimizado.

  • Un intérprete traduce y ejecuta el código línea por línea sin generar archivos intermedios, lo que permite una ejecución más dinámica pero menos eficiente.

  • Un ensamblador traduce el código fuente escrito en lenguaje ensamblador a código objeto, utilizando instrucciones de bajo nivel específicas para la máquina en cuestión.
    • Resumen
    El código fuente es escrito por programadores en un editor de texto usando un lenguaje de programación de alto nivel los cuales contienen instrucciones para que la computadora implemente el algoritmo. Es importante crear un algoritmo en pseudocódigo antes de la codificación. El código objeto es un código intermedio no directamente ejecutable, obtenido mediante un compilador. El código ejecutable es un código máquina generado al enlazar archivos de código objeto.

    Clasificación de los lenguajes de programación

    1. Lenguajes de bajo nivel


    En este apartado tenemos dos tipos de lenguajes:


  • Lenguaje máquina Este lenguaje de programación es el más antiguo que existe. El lenguaje máquina es el único que comprenden los ordenadores. Cualquier código redactado mediante lenguajes alternativos debe ser convertido a lenguaje máquina para que un ordenador pueda leerlo. El lenguaje máquina se basa en el código binario, es decir, se escribe a base de 0 y 1.Aprender lenguaje máquina es algo muy complejo y no es útil para la creación de programas o webs. Es por esto que se han creado los demás lenguajes de bajo nivel, para poder redactar código de manera sencilla que puede ser convertido en lenguaje máquina a través de un ensamblador.

  • Lenguaje ensamblador Los lenguajes de bajo nivel están muy relacionados con el lenguaje máquina. Ambos tipos de lenguaje tienen como objetivo ser comprendidos por un ordenador. La única diferencia es que los lenguajes ensambladores están hechos para ser escritos por humanos de manera sencilla. Pese a esto, tiene que ser traducido por un programa conocido como Ensamblador para que los ordenadores puedan comprenderlo. En definitiva, el lenguajes ensamblador tiene las mismas funciones que el lenguajes máquina, pero está diseñado para poder ser escrito y comprendido con facilidad por los humanos. Los códigos son mucho más cortos e intuitivos y, además, tienden a ocupar mucho menos memoria.


    • 2. Lenguajes de medio nivel


  • Los lenguajes de nivel medio tienden a ser clasificados como lenguajes de bajo nivel porque siguen dependiendo de ensambladores para poder ser comprendidos por los ordenadores. Pese a esto, permiten llevar a cabo tareas mucho más complejas que los lenguajes de bajo nivel como el uso de funciones. Aun así, muchas tareas deben ser llevadas a cabo de manera manual, aunque hay que destacar que permiten los algoritmos de búsqueda y ordenamiento a diferencia de otros lenguajes de alto nivel. En ese sentido, los lenguajes de medio nivel están orientados hacia la programación de sistemas, por lo que no son adecuados para tareas como el diseño web. Los lenguajes de medio nivel más conocidos son C y C++.


    • 3. Lenguajes de alto nivel


  • Los lenguajes de alto nivel son los más adaptados al lenguaje humano, por lo que son los más sencillos de aprender en términos generales. Como los de medio nivel, tienen que ser traducidos a lenguaje máquina para poder ser ejecutados por un ordenador. Pese a esto, debido a las modificaciones del lenguaje en relación al código máquina, los ensambladores no son útiles para esto. Los lenguajes de alto nivel se adaptan al código máquina a través de traductores y compiladores, dependiendo de si se requiere una ejecución más rápida o menos.

    Proceso de Obtención de Código Ejecutable a Partir de Código Fuente

    El proceso de crear un código ejecutable a partir de un código fuente involucra varias etapas:

    1. Escritura del código en un lenguaje de programación de alto nivel (como C, Java o Python).
    2. En el caso de lenguajes compilados (como C o C++), el código fuente se compila con un compilador como gcc o javac.
    3. El compilador realiza varias fases como el preprocesamiento, análisis léxico, análisis sintáctico, generación de código intermedio, optimización y generación de código objeto.
    4. El enlazador combina los archivos objeto generados por el compilador y resuelve las dependencias de funciones y bibliotecas externas, generando el archivo ejecutable final.
    5. El sistema operativo carga el archivo ejecutable en la memoria y lo ejecuta.

    El archivo ejecutable resultante puede tener diferentes extensiones dependiendo del sistema operativo, como .exe en Windows o .bin y .elf en sistemas Unix.

    Resumen
    El proceso de crear un código ejecutable incluye escribir en un lenguaje de programación, compilar con un compilador, enlazar archivos objeto, cargar en memoria y ejecutar en el sistema operativo.

    Concepto de Máquina Virtual y su Utilidad

    Una máquina virtual (VM) es un ordenador que opera completamente a través de software en lugar de hardware físico. Las máquinas virtuales imitan la operación de un sistema operativo o un ordenador diferente utilizando software en un ordenador físico.

    Un solo ordenador puede tener varias máquinas virtuales con sistemas operativos diferentes y funciones independientes. Esto permite, por ejemplo, ejecutar un sistema macOS en un ordenador con Windows o viceversa.

    Las máquinas virtuales son útiles para empresas y desarrolladores al disminuir costos y mejorar el funcionamiento de la red, especialmente al ofrecer software en la nube o garantizar la protección de servidores. También son una herramienta útil para diseñadores, desarrolladores y testers que necesitan probar software en diferentes entornos sin adquirir hardware adicional.



    Máquina virtual Java

    Proceso de ejecución de un programa Java:


  • 1. Código fuente (source code): Un programa Java comienza como un archivo de texto plano con extensión .java. Este código está escrito en el lenguaje de programación Java. Compilación (compilation): El compilador de Java (javac) toma este código fuente y lo convierte en un conjunto de instrucciones intermedias llamadas bytecodes, que se almacenan en archivos con extensión .class.

  • 2. Ejecución en la JVM: La JVM es responsable de leer estos bytecodes y ejecutarlos. Lo hace mediante un proceso llamado interpretación y, en algunas JVMs modernas, también compilación just-in-time (JIT), que convierte los bytecodes en código máquina específico para el sistema operativo y la arquitectura de hardware donde se está ejecutando el programa.

    • Resumen
    Una máquina virtual es un ordenador que funciona a través de software en lugar de hardware físico, imitando sistemas operativos diferentes en un solo ordenador. Son útiles para reducir costos y mejorar el funcionamiento de la red, permitiendo ejecutar sistemas diferentes en un mismo equipo. Ayudan a empresas, desarrolladores y testers a probar software sin necesidad de hardware adicional.

    Razón de ser de la ingeniería de software

    La ingeniería de software surgió como una solución para abordar los problemas que enfrentaban los proyectos de desarrollo de software a gran escala. Antes de su formalización, era común que los proyectos se excedieran en los plazos de entrega, los presupuestos o no cumplieran con los estándares esperados de calidad. Esto generaba una crisis en la industria tecnológica, conocida como la "crisis del software", donde la demanda de sistemas complejos superaba la capacidad para desarrollarlos de manera efectiva.

    La importancia de la ingeniería de software radica en que no solo se centra en la programación, sino también en la gestión y optimización de todo el ciclo de vida del software. Este ciclo incluye actividades como la planificación, el análisis de requisitos, el diseño, la implementación, las pruebas y el mantenimiento. Al aplicar principios de ingeniería, se busca garantizar que el software sea fiable, eficiente, escalable y fácil de mantener, adaptándose a las necesidades de los usuarios y a los cambios tecnológicos.

    Fases del desarrollo de una aplicación informática

    El desarrollo de software sigue un conjunto de fases bien definidas que permiten organizar el trabajo de manera sistemática, mejorar la calidad del producto final y reducir riesgos como el incumplimiento de los requisitos o la generación de errores críticos. Fases principales:

    • Análisis de Requisitos: En esta fase se identifican las necesidades específicas de los usuarios y del cliente. Es fundamental para definir qué funciones debe cumplir el software y establecer los límites del proyecto.
    • Diseño: Se desarrolla una planificación detallada que abarca desde la arquitectura del sistema hasta las interfaces de usuario. Se decide cómo se conectarán los componentes y se prioriza la escalabilidad y la facilidad de mantenimiento.
    • Codificación: Aquí los desarrolladores escriben el código que implementa el diseño establecido. Se utilizan lenguajes de programación y herramientas específicas para dar vida al software.
    • Pruebas: El software es sometido a pruebas exhaustivas para verificar que cumple con los requisitos y que no contiene errores graves. Esto incluye pruebas unitarias, de integración, de sistema y de aceptación.
    • Documentación: Se crean manuales de usuario, guías técnicas y documentación interna que facilitan la comprensión del sistema tanto para los usuarios como para el equipo de desarrollo.
    • Mantenimiento: Una vez entregado el software, es necesario realizar ajustes, resolver errores detectados en el uso real, y adaptarlo a cambios en los requisitos o en el entorno tecnológico. El mantenimiento asegura la funcionalidad y longevidad del producto.

    Perfiles que Intervienen en el Desarrollo de Software

    El desarrollo de software involucra a distintas personas con diferentes perfiles. Cada perfil contribuye de manera única a las distintas fases del proyecto, desde el análisis inicial hasta la entrega y el mantenimiento del producto. Diferentes perfiles que intervienen en el proceso:

    • Analista de Sistemas: Se encarga de analizar y definir los requisitos funcionales y técnicos del sistema en base a las necesidades del cliente o usuario final. Documenta de forma detallada los procesos y especificaciones, sirviendo como puente de comunicación entre el cliente y el equipo de desarrollo.
    • Diseñador de Software: Diseña la arquitectura del sistema, estableciendo cómo los diferentes componentes interactuarán entre sí. Define el flujo de datos, la estructura del software y selecciona las tecnologías más adecuadas para garantizar que el sistema sea eficiente, escalable y fácil de mantener.
    • Desarrollador: Es el encargado de programar el software, escribiendo y depurando el código fuente. Trabaja para implementar las funcionalidades requeridas, integrar los distintos módulos y corregir errores que puedan surgir durante el desarrollo. También participa en el mantenimiento y actualización del software una vez entregado.
    • Tester/QA: Realiza pruebas exhaustivas para garantizar que el software funcione correctamente y cumpla con los requisitos especificados. Identifica errores, inconsistencias o posibles mejoras, asegurando así la calidad y la experiencia del usuario antes de la entrega final.
    • Administrador de Bases de Datos: Gestiona las bases de datos del sistema, asegurando su correcto funcionamiento, integridad y seguridad. Es responsable de optimizar consultas, realizar copias de seguridad y garantizar la disponibilidad de los datos en todo momento.
    • Especialista en Seguridad: Se enfoca en proteger el software frente a vulnerabilidades y amenazas externas. Implementa medidas de seguridad, realiza auditorías y pruebas para garantizar que el sistema sea robusto frente a posibles ataques.
    • Usuarios: Son las personas que utilizarán el software en su día a día. Su feedback es crucial durante todo el ciclo de desarrollo para ajustar funcionalidades, mejorar la usabilidad y asegurar que el producto cumpla con sus expectativas y necesidades reales.

    Modelos de ciclo de vida del software

    Los modelos de ciclo de vida del software proporcionan una estructura para planificar, organizar y controlar el proceso de desarrollo. Elegir el modelo adecuado depende de factores como la claridad de los requisitos, la complejidad del proyecto y la necesidad de adaptabilidad.

    • Modelo en Cascada: Uso: Proyectos con requisitos claros y bien definidos desde el inicio. Importancia: Es el modelo más básico y establece una estructura sencilla y secuencial para el desarrollo. Ventajas: Simplicidad y claridad en el proceso, ideal para proyectos pequeños y bien entendidos. Limitaciones: Dificultad para adaptarse a cambios en los requisitos.
    • Modelo Incremental: Uso: Proyectos que requieren entregas funcionales tempranas y retroalimentación continua. Importancia: Introduce modularidad y permite desarrollar software usable más rápidamente. Ventajas: Reducción del riesgo al dividir el trabajo en partes más pequeñas. Limitaciones: Puede ser complejo integrar módulos si no se planifica bien.
    • Modelo Ágil: Uso: Proyectos donde los requisitos son inciertos o cambian con frecuencia. Importancia: Es el enfoque más popular en la industria actual debido a su flexibilidad y enfoque colaborativo. Ventajas: Incrementos funcionales rápidos, retroalimentación continua, y adaptación a cambios. Limitaciones: Requiere alta comunicación y compromiso constante del equipo y cliente.
    • Modelo DevOps: Uso: Proyectos que requieren una integración continua de desarrollo y operaciones para acelerar la entrega. Importancia: Es crucial para empresas que desean mantener un flujo continuo de entrega y retroalimentación. Ventajas: Automatización, colaboración mejorada y ciclos de entrega más rápidos. Limitaciones: Requiere un alto nivel de experiencia técnica y herramientas avanzadas.
    • Modelo en Espiral: Uso: Proyectos grandes, complejos o de alto riesgo. Importancia: Introduce la gestión de riesgos como parte esencial del desarrollo. Ventajas: Reduce riesgos al revisar el proyecto en iteraciones planificadas. Limitaciones: Requiere tiempo y experiencia en análisis de riesgos.
    • Modelo RAD (Rapid Application Development): Uso: Proyectos con requisitos dinámicos y necesidad de plazos cortos. Importancia: Se utiliza para acelerar el desarrollo mediante prototipos y componentes reutilizables. Ventajas: Rápida respuesta a los cambios en los requisitos y menor tiempo de entrega. Limitaciones: Es menos efectivo para proyectos grandes o con restricciones técnicas.

    Metodologías de desarrollo de software actuales

    En la actualidad, las metodologías ágiles son ampliamente utilizadas por su enfoque en la colaboración, la adaptabilidad y la entrega rápida de valor al cliente. Estas metodologías permiten a los equipos responder rápidamente a los cambios y entregar productos de alta calidad de manera iterativa.

    • Scrum: Es un marco ágil que organiza el trabajo en ciclos llamados sprints, con una duración típica de 2 a 4 semanas.
      • Durante cada sprint, el equipo trabaja en un conjunto de tareas previamente priorizadas por el Product Owner.
      • El equipo se reúne diariamente en reuniones cortas llamadas *daily stand-ups* para sincronizarse y resolver bloqueos.
      • Al finalizar el sprint, se lleva a cabo una revisión para presentar el trabajo terminado y una retrospectiva para identificar mejoras en el proceso.
      • Este enfoque ayuda a entregar incrementos funcionales de manera regular, lo que permite ajustes rápidos según la retroalimentación recibida.
    • Kanban: Es una metodología ágil que utiliza un sistema visual de gestión del flujo de trabajo para optimizar los procesos.
      • Se basa en un tablero dividido en columnas que representan el estado de las tareas, como "Por hacer", "En progreso" y "Completado".
      • El equipo limita la cantidad de tareas en cada columna para evitar sobrecargas y mejorar la eficiencia.
      • Es especialmente útil para proyectos que no tienen plazos rígidos y necesitan un flujo continuo de trabajo en lugar de ciclos cerrados.
      • Proporciona una vista clara del progreso y ayuda a identificar problemas, como cuellos de botella, para implementar mejoras rápidamente.
