Desarrollo De Software

Concepto de Software

El software es el conjunto de programas, aplicaciones y datos que permiten a una computadora o dispositivo realizar tareas específicas.

En esto tenemos el Software de aplicación, que son los programas diseñados para realizar tareas específicas para el usuario, como procesadores de texto, navegadores web, aplicaciones de diseño, juegos, etc. Luego tendríamos el Software de sistema que incluye los programas que permiten que el hardware funcione correctamente, como los sistemas operativos (por ejemplo, Windows, macOS, Linux), controladores de dispositivos, y utilidades del sistema, y por último el Software de desarrollo que son las herramientas que permiten a los programadores crear otros programas, como lenguajes de programación (Java, Python, C++) y entornos de desarrollo integrados (IDEs).

Resumen
El software es esencial para el funcionamiento de los dispositivos electrónicos, ya que da instrucciones precisas sobre cómo debe ejecutarse cada operación o proceso.

Conceptos de Código Fuente, Código Objeto y Código Ejecutable

Código Fuente

El código fuente es el escrito por los programadores en un editor de texto, utilizando un lenguaje de programación de alto nivel. Contiene las instrucciones para que la computadora implemente el algoritmo. Es importante crear un algoritmo previamente en pseudocódigo para agilizar la posterior codificación en un lenguaje de programación. Para obtener el código fuente de una aplicación, se deben seguir varias etapas, como el análisis, diseño, elección del lenguaje de programación y la codificación del algoritmo.

Código Objeto

El código objeto se trata de un código intermedio para los lenguajes intermedios, por lo tanto no es directamente ejecutable por el equipo. Para que la máquina pueda ejecutar el programa es necesario transformarlo al lenguaje que ésta maneja con un intérprete. El código objeto se obtiene mediante el uso de un compilador.

Código Ejecutable

El código ejecutable es un código máquina que se genera al enlazar los archivos de código objeto con rutinas y bibliotecas necesarias. El sistema operativo carga este código en la memoria RAM y lo ejecuta. Los programas compilados no producen código objeto, ya que el proceso de compilación convierte directamente de fuente a ejecutable.

Resumen
El código fuente es escrito por programadores en un editor de texto usando un lenguaje de programación de alto nivel los cuales contienen instrucciones para que la computadora implemente el algoritmo. Es importante crear un algoritmo en pseudocódigo antes de la codificación. El código objeto es un código intermedio no directamente ejecutable, obtenido mediante un compilador. El código ejecutable es un código máquina generado al enlazar archivos de código objeto.

Lenguajes de Programación (Más Relevantes)

JavaScript

JavaScript es usado por el 98% de las webs de todo el mundo. Junto a HTML y CSS, es uno de los pilares esenciales de internet. Originalmente, era utilizado exclusivamente en navegadores, pero actualmente se ha integrado como parte fundamental de algunos servidores y una gran variedad de aplicaciones.

Python

Python está presente en servicios populares como Spotify, Instagram y Netflix. Se caracteriza por su enfoque multiparadigma y una alta legibilidad de su código. Fue creado a finales de los años 80 por Guido van Rossum y ha perdurado en el tiempo como uno de los lenguajes más populares.

Java

Java está presente en multitud de dispositivos y sistemas, como teléfonos móviles, ordenadores, centros de datos, consolas de videojuegos y aplicaciones basadas en la comunicación cliente-servidor en la web.

C#

C# fue creado por Microsoft como parte de su plataforma .NET. Su nombre hace referencia al Do sostenido, una metáfora con la que sus creadores subrayan su superioridad en la escala de la programación con respecto a su antecesor, C++.

SQL

SQL (Structured Query Language) es un lenguaje de consulta de bases de datos ampliamente utilizado, especialmente en MySQL, un software de gestión de bases de datos relacionales.

Resumen
JavaScript, HTML y CSS son fundamentales en internet. Python se destaca por su legibilidad y es usado en sitios como Spotify e Instagram. Proceso de Obtención de Código Ejecutable a Partir de Código Fuente

El proceso de crear un código ejecutable a partir de un código fuente involucra varias etapas:

  1. Escritura del código en un lenguaje de programación de alto nivel (como C, Java o Python).
  2. En el caso de lenguajes compilados (como C o C++), el código fuente se compila con un compilador como gcc o javac.
  3. El compilador realiza varias fases como el preprocesamiento, análisis léxico, análisis sintáctico, generación de código intermedio, optimización y generación de código objeto.
  4. El enlazador combina los archivos objeto generados por el compilador y resuelve las dependencias de funciones y bibliotecas externas, generando el archivo ejecutable final.
  5. El sistema operativo carga el archivo ejecutable en la memoria y lo ejecuta.

El archivo ejecutable resultante puede tener diferentes extensiones dependiendo del sistema operativo, como .exe en Windows o .bin y .elf en sistemas Unix.

Resumen
El proceso de crear un código ejecutable incluye escribir en un lenguaje de programación, compilar con un compilador, enlazar archivos objeto, cargar en memoria y ejecutar en el sistema operativo.

Concepto de Máquina Virtual y su Utilidad

Una máquina virtual (VM) es un ordenador que opera completamente a través de software en lugar de hardware físico. Las máquinas virtuales imitan la operación de un sistema operativo o un ordenador diferente utilizando software en un ordenador físico.

Un solo ordenador puede tener varias máquinas virtuales con sistemas operativos diferentes y funciones independientes. Esto permite, por ejemplo, ejecutar un sistema macOS en un ordenador con Windows o viceversa.

Las máquinas virtuales son útiles para empresas y desarrolladores al disminuir costos y mejorar el funcionamiento de la red, especialmente al ofrecer software en la nube o garantizar la protección de servidores. También son una herramienta útil para diseñadores, desarrolladores y testers que necesitan probar software en diferentes entornos sin adquirir hardware adicional.

Resumen
Una máquina virtual es un ordenador que funciona a través de software en lugar de hardware físico, imitando sistemas operativos diferentes en un solo ordenador. Son útiles para reducir costos y mejorar el funcionamiento de la red, permitiendo ejecutar sistemas diferentes en un mismo equipo. Ayudan a empresas, desarrolladores y testers a probar software sin necesidad de hardware adicional.

Razón de ser de la ingeniería de software

La ingeniería de software surgió como una solución para abordar los problemas que enfrentaban los proyectos de desarrollo de software a gran escala. Antes de su formalización, era común que los proyectos se excedieran en los plazos de entrega, los presupuestos o no cumplieran con los estándares esperados de calidad. Esto generaba una crisis en la industria tecnológica, conocida como la "crisis del software", donde la demanda de sistemas complejos superaba la capacidad para desarrollarlos de manera efectiva.

La importancia de la ingeniería de software radica en que no solo se centra en la programación, sino también en la gestión y optimización de todo el ciclo de vida del software. Este ciclo incluye actividades como la planificación, el análisis de requisitos, el diseño, la implementación, las pruebas y el mantenimiento. Al aplicar principios de ingeniería, se busca garantizar que el software sea fiable, eficiente, escalable y fácil de mantener, adaptándose a las necesidades de los usuarios y a los cambios tecnológicos.

Fases del desarrollo de una aplicación informática

El desarrollo de software sigue un conjunto de fases bien definidas que permiten organizar el trabajo de manera sistemática, mejorar la calidad del producto final y reducir riesgos como el incumplimiento de los requisitos o la generación de errores críticos. Fases principales:

  • Análisis de Requisitos: En esta fase se identifican las necesidades específicas de los usuarios y del cliente. Es fundamental para definir qué funciones debe cumplir el software y establecer los límites del proyecto.
  • Diseño: Se desarrolla una planificación detallada que abarca desde la arquitectura del sistema hasta las interfaces de usuario. Se decide cómo se conectarán los componentes y se prioriza la escalabilidad y la facilidad de mantenimiento.
  • Codificación: Aquí los desarrolladores escriben el código que implementa el diseño establecido. Se utilizan lenguajes de programación y herramientas específicas para dar vida al software.
  • Pruebas: El software es sometido a pruebas exhaustivas para verificar que cumple con los requisitos y que no contiene errores graves. Esto incluye pruebas unitarias, de integración, de sistema y de aceptación.
  • Mantenimiento: Una vez entregado el software, es necesario realizar ajustes, resolver errores detectados en el uso real, y adaptarlo a cambios en los requisitos o en el entorno tecnológico.

Perfiles que Intervienen en el Desarrollo de Software

El desarrollo de software involucra a distintas personas con diferentes perfiles. Cada perfil contribuye de manera única a las distintas fases del proyecto, desde el análisis inicial hasta la entrega y el mantenimiento del producto. Diferentes perfiles que intervienen en el proceso:

  • Analista de Sistemas: Se encarga de analizar y definir los requisitos funcionales y técnicos del sistema en base a las necesidades del cliente o usuario final. Documenta de forma detallada los procesos y especificaciones, sirviendo como puente de comunicación entre el cliente y el equipo de desarrollo.
  • Diseñador de Software: Diseña la arquitectura del sistema, estableciendo cómo los diferentes componentes interactuarán entre sí. Define el flujo de datos, la estructura del software y selecciona las tecnologías más adecuadas para garantizar que el sistema sea eficiente, escalable y fácil de mantener.
  • Desarrollador: Es el encargado de programar el software, escribiendo y depurando el código fuente. Trabaja para implementar las funcionalidades requeridas, integrar los distintos módulos y corregir errores que puedan surgir durante el desarrollo. También participa en el mantenimiento y actualización del software una vez entregado.
  • Tester/QA: Realiza pruebas exhaustivas para garantizar que el software funcione correctamente y cumpla con los requisitos especificados. Identifica errores, inconsistencias o posibles mejoras, asegurando así la calidad y la experiencia del usuario antes de la entrega final.
  • Administrador de Bases de Datos: Gestiona las bases de datos del sistema, asegurando su correcto funcionamiento, integridad y seguridad. Es responsable de optimizar consultas, realizar copias de seguridad y garantizar la disponibilidad de los datos en todo momento.
  • Especialista en Seguridad: Se enfoca en proteger el software frente a vulnerabilidades y amenazas externas. Implementa medidas de seguridad, realiza auditorías y pruebas para garantizar que el sistema sea robusto frente a posibles ataques.
  • Usuarios: Son las personas que utilizarán el software en su día a día. Su feedback es crucial durante todo el ciclo de desarrollo para ajustar funcionalidades, mejorar la usabilidad y asegurar que el producto cumpla con sus expectativas y necesidades reales.

Modelos de ciclo de vida del software

Los modelos de ciclo de vida del software proporcionan una estructura para planificar, organizar y controlar el proceso de desarrollo. Elegir el modelo adecuado depende de factores como la claridad de los requisitos, la complejidad del proyecto y la necesidad de adaptabilidad.

  • Modelo en Cascada: Es un modelo lineal y secuencial, donde cada fase debe completarse antes de comenzar la siguiente. Es ideal para proyectos con requisitos bien definidos y poco propensos a cambios.
  • Modelo Incremental: Divide el proyecto en módulos que se desarrollan y entregan por iteraciones. Esto permite obtener retroalimentación temprana y reducir el tiempo de espera para obtener funcionalidades utilizables.
  • Modelo Ágil: Promueve ciclos de desarrollo cortos e iterativos, conocidos como sprints, donde los equipos entregan incrementos funcionales del software. Es flexible y se adapta a cambios frecuentes en los requisitos.
  • Modelo DevOps: Integra el desarrollo y las operaciones, automatizando tareas como las pruebas y el despliegue. Esto acelera la entrega de software y mejora la calidad mediante la colaboración continua entre equipos.

Metodologías de desarrollo de software actuales

En la actualidad, las metodologías ágiles son ampliamente utilizadas por su enfoque en la colaboración, la adaptabilidad y la entrega rápida de valor al cliente. Estas metodologías permiten a los equipos responder rápidamente a los cambios y entregar productos de alta calidad de manera iterativa.

  • Scrum: Divide el trabajo en ciclos llamados sprints, que suelen durar de 2 a 4 semanas. Cada sprint tiene objetivos específicos y al final se realiza una revisión para mejorar en el siguiente ciclo.
  • Kanban: Utiliza tableros visuales para gestionar el flujo de trabajo, permitiendo a los equipos identificar cuellos de botella y optimizar procesos. Es ideal para proyectos que requieren un enfoque más flexible y continuo.

Por otro lado, DevOps combina prácticas de desarrollo y operaciones para automatizar y optimizar procesos como las pruebas, la integración y el despliegue continuo. Esto no solo mejora la velocidad de entrega, sino también la estabilidad y calidad del software. Estas metodologías modernas están transformando la manera en que las organizaciones crean y mantienen software, haciéndolo más centrado en las necesidades del usuario final.
